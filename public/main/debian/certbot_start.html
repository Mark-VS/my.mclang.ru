Создаю сертификат для четырёх доменов: test01.mclang.ru, www.test01.mclang.ru, test02.mclang.ru и www.test02.mclang.ru:

Выполнять нужно с запущенным Nginx, т.к. получение и обновление сертификатов происходит через Nginx.

sudo certbot --nginx -d test01.mclang.ru -d www.test01.mclang.ru -d test02.mclang.ru -d www.test02.mclang.ru

Сертификаты создались. Но certbot ругнулся на то, что указаны не все блоки. Он нашёл только test01.mclang.ru, а остальные домены я не указал в конфиге.

Сертификат установился сюда:

  • /etc/letsencrypt/live/test01.mclang.ru/fullchain.pem
  • /etc/letsencrypt/live/test01.mclang.ru/privkey.pem

sudo ls -l /etc/letsencrypt/live/test01.mclang.ru
  # total 4
  # lrwxrwxrwx 1 root root  40 Apr  8 19:56 cert.pem -> ../../archive/test01.mclang.ru/cert1.pem
  # lrwxrwxrwx 1 root root  41 Apr  8 19:56 chain.pem -> ../../archive/test01.mclang.ru/chain1.pem
  # lrwxrwxrwx 1 root root  45 Apr  8 19:56 fullchain.pem -> ../../archive/test01.mclang.ru/fullchain1.pem
  # lrwxrwxrwx 1 root root  43 Apr  8 19:56 privkey.pem -> ../../archive/test01.mclang.ru/privkey1.pem
  # -rw-r--r-- 1 root root 692 Apr  8 19:56 README

Обратим внимание, что права на папку /etc/letsencrypt/live ограничены:

sudo ls -ld /etc/letsencrypt/live
  # drwx------ 3 root root 4096 Apr  8 19:56 /etc/letsencrypt/live

Но я права не менял. Nginx запускается от sudo, а значит, может читать файлы из этой папки.

Добавим в наш минимальный конфиг и секцию для TLS со ссылками на сертификат и ключ:

server {
   listen 80;
   listen [::]:80;

   server_name   test01.mclang.ru;
   root   /var/www/html;
   index index.html index.htm index.nginx-debian.html;
   location / {
      try_files $uri $uri/ =404;
   }
}
server {
   listen 443 ssl;
   listen [::]:443 ssl;
   ssl_certificate /etc/letsencrypt/live/test01.mclang.ru/fullchain.pem;
   ssl_certificate_key /etc/letsencrypt/live/test01.mclang.ru/privkey.pem;

   server_name   test01.mclang.ru;
   root   /var/www/html;
   index index.html index.htm index.nginx-debian.html;
   location / {
      try_files $uri $uri/ =404;
   }
}

Можно улучшить конфиг так, чтобы пользователя с http перекидывало на https. Таким образом, пользователь сможет зайти только через безопасное соединение:

server {
   listen 80;
   listen [::]:80;
   server_name   test01.mclang.ru;

   return 301 https://$host$request_uri;
}
server {
   listen 443 ssl;
   listen [::]:443 ssl;
   ssl_certificate /etc/letsencrypt/live/test01.mclang.ru/fullchain.pem;
   ssl_certificate_key /etc/letsencrypt/live/test01.mclang.ru/privkey.pem;

   server_name   test01.mclang.ru;
   root   /var/www/html;
   index index.html index.htm index.nginx-debian.html;
   location / {
      try_files $uri $uri/ =404;
   }
}

Чтобы ещё дополнительно обработать адреса с www, добавим их в server_name:

server {
   listen 80;
   listen [::]:80;
   server_name   test01.mclang.ru www.test01.mclang.ru;

   return 301 https://$host$request_uri;
}
server {
   listen 443 ssl;
   listen [::]:443 ssl;
   ssl_certificate /etc/letsencrypt/live/test01.mclang.ru/fullchain.pem;
   ssl_certificate_key /etc/letsencrypt/live/test01.mclang.ru/privkey.pem;

   server_name   test01.mclang.ru www.test01.mclang.ru;
   root   /var/www/html;
   index index.html index.htm index.nginx-debian.html;
   location / {
      try_files $uri $uri/ =404;
   }
}
