Node.js

Официальная страница DEB-репозитория Node.js: https://deb.nodesource.com/

Устанавливаем Node.js 22.x LTS

~$ curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash - 
~$ sudo apt-get install -y nodejs

# Проверяем:
node -v
  # v22.22.2
npm -v
  # 10.9.7
  • curl -fsSL https://deb.nodesource.com/setup_22.x → скачивает скрипт установки Node.js 22 LTS.

  • | sudo bash - → сразу выполняет этот скрипт от root. Скрипт:

    • добавляет репозиторий NodeSource для Node 22,
    • добавляет GPG-ключ,
    • обновляет индексы пакетов (apt update).
  • sudo apt-get install -y nodejs → устанавливает Node.js и npm.

Установка Yarn

Есть три способа установки Yarn на Debian:

  • Через официальный Yarn-репозиторий для Debian
  • Через npm
  • Через Corepack (новый способ от Node.js). Corepack — это менеджер пакетов, встроенный в новые Node.js версии (начиная с Node 16+), который умеет запускать yarn и pnpm.

Ставим Yarn через официальный Yarn-репозиторий для Debian:

# Скачиваем публичный ключ Yarn и сохраняем его в правильное место
curl -fsSL https://dl.yarnpkg.com/debian/pubkey.gpg | sudo tee /usr/share/keyrings/yarn-archive-keyring.gpg >/dev/null

# Добавляем репозиторий Yarn с использованием keyring
echo "deb [signed-by=/usr/share/keyrings/yarn-archive-keyring.gpg] https://dl.yarnpkg.com/debian/ stable main" | sudo tee /etc/apt/sources.list.d/yarn.list

# Обновляем списки пакетов:
sudo apt update

# Ставим Yarn:
sudo apt install yarn -y

# Проверяем:
yarn --version
  # 1.22.22

Если устанавливать yarnpkg из официального репозитория Debian, то он у меня конфликтовал со свежим Node.js.
