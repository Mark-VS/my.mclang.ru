ГПТ также советует поставить (но вроде бы как не обязательно). Я их не устанавливал.

sudo apt install build-essential -y
sudo apt install pkg-config -y
  • build-essenial – Это метапакет, который устанавливает всё необходимое для сборки программ. build-essential включает:
    • gcc — компилятор C
    • g++ — компилятор C++
    • make — система сборки
    • стандартные библиотеки (libc6-dev)
    • другие базовые инструменты
  • pkg-config – это помощник компилятора. Он говорит, где лежат библиотеки и как их подключить. Часто нужен при сборке сложных проектов. Просто “ставим на всякий случай”, чтобы не ловить ошибки.

Запустил сборку:

Но получил сообщение о том, что не хватает места на диске.

Когда я собираю GoToSocial, временные файлы пишутся в /tmp.

А у меня на Debian /tmp – это файловая система tmpfs, т.е. временная фс в оперативной памяти (RAM). То есть /tmp – это не диск, это память компьютера.

lsblk
  # ...
  # tmpfs                450M  274M  176M  61% /tmp
  # ...

Есть переменная TMPDIR, откуда программы узнают о том, куда писать временные файлы. Если эта переменная не задана, то по умолчанию временные файлы пишутся в /tmp. У меня TMPDIR не задана. Поэтому, если я хочу, чтобы Go при сборке писал файлы на диск, то можно просто задать нужный путь в переменной TMPDIR:

export TMPDIR=~/tmp

# А теперь можно и собрать:
go build -x

У меня сборка заняла чуть больше 30-ти минут. Примерно 33. И появился бинарник gotosocial.

Также появилась папка ~/go/pkg/. Это кэш Go. Сюда Go кладёт промежуточные пакеты во время сборки. По умолчанию кэш кладётся в GOPATH/pkg, а переменная GOPATH по умолчанию равна ~/go.

Переносим бинарник в /usr/local/bin/:

sudo mv ./gotosocial /usr/local/bin/

# Проверяем:
which gotosocial

Теперь создаём системного пользователя gotosocial, чтобы от его имени запускать GoToSocialf:

sudo useradd -r -m -d /var/lib/gotosocial -s /usr/sbin/nologin gotosocial

# Проверяем:
getent passwd gotosocial
  # gotosocial:x:998:996::/var/lib/gotosocial:/usr/sbin/nologin

Копируем собранный фронтенд и шаблоны в доманюю директорию:

# Создаём папку web:
sudo mkdir -p /var/lib/gotosocial/web

# Копируем в неё шаблоны и ассеты:
sudo cp -r ~/Downloads/gotosocial/web/template /var/lib/gotosocial/web/
sudo cp -r ~/Downloads/gotosocial/web/assets /var/lib/gotosocial/web/

# Проверяем права и владельца:
sudo ls -ld /var/lib/gotosocial/web
  # drwxr-xr-x 7 root root 4096 Apr  9 22:41 /var/lib/gotosocial/web

# Меняем владельца на gotosocial:
sudo chown -R gotosocial:gotosocial /var/lib/gotosocial/web
# Запрещаем доступ остальным:
sudo chmod -R 750 /var/lib/gotosocial/web

# Проверяем права ещё раз:
sudo ls -ld /var/lib/gotosocial/web
  # drwxr-x--- 7 gotosocial gotosocial 4096 Apr  9 22:41 /var/lib/gotosocial/web

У меня у папки /var/lib/gotosocial права - 700, а у assets и у template внутри - права 750. Впринципе, можно всем папкам внутри поменять права на 700.

Создаём конфиг:


sudo mkdir -p /etc/gotosocial
sudo touch /etc/gotosocial/config.yaml

sudo chown root:gotosocial /etc/gotosocial/config.yaml
sudo chmod 640 /etc/gotosocial/config.yaml
  • владелец: root → только root может менять файл
  • группа: gotosocial → сервис может читать

  • права 640 →

    • root → rw
    • gotosocial → r
    • остальные → ничего


web-template-dir: "/var/lib/gotosocial/template/"
web-asset-base-dir: "/var/lib/gotosocial/web/assets/"
