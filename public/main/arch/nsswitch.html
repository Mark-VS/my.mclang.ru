nsswitch.conf

Дефолтный /etc/nsswitch.conf в Arch:

# Name Service Switch configuration file.
# See nsswitch.conf(5) for details.

passwd: files systemd
group: files [SUCCESS=merge] systemd
shadow: files systemd
gshadow: files systemd

publickey: files

hosts: files mdns_minimal [NOTFOUND=return] dns
networks: files

protocols: files
services: files
ethers: files
rpc: files

Для начала хотелось бы рассмотреть строку: hosts: files mdns_minimal [NOTFOUND=return] dns

  • files – это модуль NSS, который означает: использовать файлы из /etc/

    Для hosts это значит: использовать /etc/hosts.

    Для passwd это значит: использовать /etc/passwd. Для group – /etc/group. И т.д.

    То есть, files – это имя модуля в glibc: libnss_files.so

  • dns – на самом деле имя модуля в glibc: libnss_dns.so.
  • mdns_minimal – это упрощённый обработчик mDNS (Multicast DNS). mDNS = локальный DNS без сервера. Обрабатывает только домены вида *.local

  • [NOTFOUND=return] – Это условие управления потоком. Что-то вроде: “Если результат = NOTFOUND → остановиться и дальше не идти”. Что значит NOTFOUND? Сервис точно знает, что записи нет.

    Есть разные статусы:

    • SUCCESS – нашли
    • NOTFOUND – точно нет
    • UNAVAIL – сервис не работает
    • TRYAGAIN – временная ошибка

То есть, запись hosts files dns буквально означает: "Сначала вызови модуль libnss_files, а потом – libnss_dns".

Пошаговое поведение:

  1. files

    • сначала /etc/hosts
    • Если нашли → return SUCCESS, всё ок
    • Если не нашли → идём дальше

  2. mdns_minimal

    • Пытается найти только .local через локальную сеть.
    • Если найдено → ok.
    • Если NOTFOUND → [NOTFOUND=return] срабатывает.

  3. dns после [NOTFOUND=return] – влияет только на статус NOTFOUND от mdns_minimal.

    • Если mdns_minimal вернёт UNAVAIL или TRYAGAIN, поиск не остановится, и dns будет использоваться.
    • Если имя вообще не .local, mdns_minimal может просто пропустить, а не возвращать NOTFOUND. Тогда dns сработает.

    То есть dns после [NOTFOUND=return] — это резервный вариант на случай, если mDNS не сможет обработать имя или вернёт ошибку, отличную от NOTFOUND.

Добавим в /etc/dnsmasq.conf ещё три строчки:

# слушаем только нужный интерфейс
interface=enp2s0

# слушаем локалхост и IP в сети
listen-address=127.0.0.1,192.168.1.3

# привязываемся к интерфейсу (важно!)
bind-interfaces

# твоя зона
address=/gts.test/192.168.1.3

# полезные опции
domain-needed
bogus-priv
