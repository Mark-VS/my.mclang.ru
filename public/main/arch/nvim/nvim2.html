Lazy

Официальный сайт lazy.nvim: lazy.folke.io

Далее кусок кода с оф. сайта – для установки lazy.nvim:

-- Bootstrap lazy.nvim
local lazypath = vim.fn.stdpath("data") .. "/lazy/lazy.nvim"
if not (vim.uv or vim.loop).fs_stat(lazypath) then
  local lazyrepo = "https://github.com/folke/lazy.nvim.git"
  local out = vim.fn.system({ "git", "clone", "--filter=blob:none", "--branch=stable", lazyrepo, lazypath })
  if vim.v.shell_error ~= 0 then
    vim.api.nvim_echo({
      { "Failed to clone lazy.nvim:\n", "ErrorMsg" },
      { out, "WarningMsg" },
      { "\nPress any key to exit..." },
    }, true, {})
    vim.fn.getchar()
    os.exit(1)
  end
end
vim.opt.rtp:prepend(lazypath)

-- Make sure to setup `mapleader` and `maplocalleader` before
-- loading lazy.nvim so that mappings are correct.
-- This is also a good place to setup other settings (vim.opt)
vim.g.mapleader = " "
vim.g.maplocalleader = "\\"

-- Setup lazy.nvim
require("lazy").setup({

})

Пример сетапа с оф. сайта:

-- Setup lazy.nvim
require("lazy").setup({
  spec = {
    -- import your plugins
    { import = "plugins" },
  },
  -- Configure any other settings here. See the documentation for more details.
  -- colorscheme that will be used when installing plugins.
  install = { colorscheme = { "habamax" } },
  -- automatically check for plugin updates
  checker = { enabled = true },
})

Установим плагин hop (т.е. добавим его в setup):

require("lazy").setup({
   { 'phaazon/hop.nvim' }
})

И подключим файл ~/.config/nvim/lua/core/plugins.lua:

-- ~/.config/nvim/init.lua

require('core.plugins')

neo-tree

Оф. страничка neo-tree: github.com/nvim-neo-tree/neo-tree.nvim

Добавим neo-tree в setup (с черырьмя зависимостями, которые ему необходимы):

~/.config/nvim/core/plugins.lua

require("lazy").setup({
   { 'phaazon/hop.nvim' },
   {
      'nvim-neo-tree/neo-tree.nvim',
      branch = 'v3.x',
      dependencies = {
         'nvim-lua/plenary.nvim', 'nvim-tree/nvim-web-devicons',
         'MunifTanjim/nui.nvim', 's1n7ax/nvim-window-picker'
      }
   },
})

Теперь можно запустить Lazy и установить neo-tree и все его зависимости. И вот они установлены, НО не инициализированы.

И вот можно пользоватьсяe деревом файлов:

-- ~/.config/nvim/plugins/neotree.lua
:Neotree reveal left # открывает дерево слева
:Neotree close       # закрывает окно с деревом
:Neotree float       # открывает дерево во всплывающем окне

Маппинги

Для удобства создадим маппинги. Они у нас будут в файле ~/.config/nvim/lua/core/mappings.lua:

-- ~/.config/nvim/core/mappings

vim.g.mapleader = " "

-- NeoTree
vim.keymap.set('n', 'e', ':Neotree float focus')
vim.keymap.set('n', 'o', ':Neotree float git_status')

Подключим mappings.lua в init.lua:

-- ~/.config/nvim/init.lua

require('core.plugins')
require('core.mappings')

Теперь можно запустить nvim и использовать Пробел, e для открытия всплывающего окна с деревом файлов и Пробел, o – для git-статуса.

TreeSitter

Оф. страничка: github.com/nvim-treesitter/nvim-treesitter

У меня после установки treesitter доступны только пять команд:

  • :TSInstall
  • :TSInstallFromGrammar
  • :TSLog
  • :TSUninstall
  • :TSUpdate

А остальных, типа TSInstallInfo, TSBufInfo, TSToggle, нет.

Добавим в setup nvim-treesitter:

-- ~/.config/nvim/core/pligins.lua

require("lazy").setup({
   { 'phaazon/hop.nvim' },
   {
      'nvim-neo-tree/neo-tree.nvim',
      branch = 'v3.x',
      dependencies = {
         'nvim-lua/plenary.nvim', 'nvim-tree/nvim-web-devicons',
         'MunifTanjim/nui.nvim', 's1n7ax/nvim-window-picker'
      }
   },
   { 'nvim-treesitter/nvim-treesitter' },
})

И установим его, запустив Lazy.

Включить treesitter-подсветку в vim:


:lua vim.treesitter.start()   # включить подсветку
:lua vim.treesitter.stop()    # отключить подсветку

Полезные команды

:h treesitter-highlight   # посмотреть справку по подсветке
:h nvim-treesitter-commands # посмотреть доступные команды

:set filetype   # выведет тип текущего файла
:set filetype=  # присвоит пустое значение текущему типу. Подсветка уйдёт
:set filetype=lua  # присвоит обратно тип и вернёт подсветку
