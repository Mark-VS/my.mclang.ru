ss

ss – замена старому netstat


ss -t src :443

# показывает TCP-соединения, которые ...
# ... находятся в состоянии SYN-RECV:
ss -ant state syn-recv

# считает кол-во строк вывода предыдущей команды:
watch -n1 'ss -ant state syn-recv | wc -l'

# кто слушает порт 443:
sudo ss -lntp | grep 443:

# посмотреть очередь ожидания:
ss -lnt

# кто владелец сети:
whois 91.221.176.1

Покажи все TCP-соединения и TCP-сокеты, где на моей стороне используется порт 443!

ss -t src :443

  • ss – показать сокеты/соединения
  • -t – только TCP
  • src :443 – показать записи, у которых локальный (source) порт равен 443
  • -s – summary (сводка) – не показывает сами сокеты, а выводит статистику по ним
  • -l – только слушающие порты
  • -p – процессы, владеющие сокетами
  • -n – не преобразовывать номера портов в имена (443 вместо https)
