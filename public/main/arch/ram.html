ram

три способа посмотреть инфу о RAM:

free -h
grep MemTotal /proc/meminfo
sudo dmidecode -t memory

Посмотреть топ процессов по реальному потреблению памяти:

smem -rk
  # PID User     Command                         Swap      USS      PSS      RSS 
  # 7922 mark     /usr/lib/qt6/QtWebEnginePro        0   979.7M     1.0G     1.1G 
  # 7833 mark     /usr/lib/firefox/firefox           0   348.2M   394.7M   525.3M 
  # 2232 mark     /usr/bin/falkon                    0   262.3M   298.4M   390.1M 
  # 8763 mark     /usr/lib/firefox/firefox -c        0   201.9M   224.3M   317.7M 
  # 2361 mark     /usr/lib/qt6/QtWebEnginePro        0    57.4M    99.0M   172.4M 
  # 8210 mark     /usr/lib/firefox/firefox -c        0    38.3M    55.1M   142.1M 
  # 8479 mark     /usr/lib/firefox/firefox -c        0    26.1M    38.2M   114.6M 
  # ...
  • USS (Unique Set Size) — память, только для этого процесса
  • PSS (Proportional Set Size) — память процесса с учётом доли shared (лучше всего)
  • RSS (Resident Set Size) — вся физическая память процесса (включая библиотеки, разделяемые другими)

Вывести топ процессов (две колонки):

smem -krc "name pss"
  # Name                          PSS 
  # QtWebEngineProc            561.2M 
  # Isolated Web Co            512.8M 
  # firefox                    384.3M 
  # falkon                     316.9M 
  # QtWebEngineProc            142.2M 
  # QtWebEngineProc            107.2M 
  # Privileged Cont             85.2M
  # ...
  • -k – abbreviate (сокращённая запись), т. е. KB и MB вместо байтов. То есть просто "читабельный вывод"
  • -r – обратная сортировка (т.е. от большего к меньшему)
  • -c – columns, говорит какие колонки отобразить
