GoToSocial с Codeberg

Официальный сайт: https://gotosocial.org/

Страничка на Codeberg: https://codeberg.org/superseriousbusiness/gotosocial

Документация: https://docs.gotosocial.org/en/stable/

Repology: https://repology.org/project/gotosocial/information

Скачиваем GoToSocial с Codeberg.org:

git clone https://codeberg.org/superseriousbusiness/gotosocial
cd gotosocial

# Либо можно скачать исходники в .zip- или в .tar.gz-формате
# ... через браузер.
# И распаковываем их:
unzip gotosocial-main.zip 
# Или:
tar -xzvf gotosocial-main.tar.gz

# нам нужно найти файл main.go:
# Он находится здесь: cmd/gotosocial/main.go

# Запускаем компиляцию:
cd cmd/gotosocial/
go build

# Если нужно показать, что именно делает Go:
go build -x

# Или из основной директории: 
go build -o gotosocial ./cmd/gotosocial

# Появился бинарник gotosocial:
ls -l gotosocial
  # -rwxr-xr-x 1 mark mark 136726647 Mar 23 08:37 gotosocial

Перемещаем бинарник в систему:

sudo mv gotosocial /usr/local/bin/

# Проверка:
gotosocial --help

Теперь создадим системного пользователя gotosocial:

sudo useradd --system \
  --home /var/lib/gotosocial \
  --shell /usr/sbin/nologin
  gotosocial

# Создаём директории:
sudo mkdir -p /var/lib/gotosocial
sudo mkdir -p /etc/gotosocial

# Папки создались от root'а, поэтому владелец и группа у них: root
# Меняем владельца домашней папки на gotosocial:
sudo chown -R gotosocial:gotosocial /var/lib/gotosocial/

# Создаём конфиг:
sudo touch /etc/gotosocial/config.yaml

Конфигурационный файл

Дефолтный конфигурационный файл лежит в папке с исходниками в example/config.yaml.

Этот конфиг очень раздут из-за огромного количества подробных комментариев.

Полный дефолтный конфиг БЕЗ КОММЕНТАРИЕВ можно посмотреть здесь!

Минимальный рабочий конфиг, который предложил гпт:

host: "your-domain.com"
account-domain: "your-domain.com"

# port: 8080

db-type: "sqlite"
db-address: "/var/lib/gotosocial/gotosocial.db"

web-template-base-dir: "/var/lib/gotosocial/web/template/"
web-asset-base-dir: "/var/lib/gotosocial/web/assets/"

# storage-backend: "local"
storage-local-base-path: "/var/lib/gotosocial/storage"

У GoToSocial нет определённого места для конфига в домашней директории пользователя. Пути очень flexible и зависят от того, как был запущен GoToSocial. Основной способ задания пути для конфига – это использовать при запуске приложения опцию --config-path или переменную окружения GTS_CONFIG_PATH.

Если мы будем следовать привычной логике других приложений, то системный конфиг должен быть в /etc/gotosocial/config.yaml, а пользовательский – в $HOME/.config/gotosocial/config.yaml, то есть, в нашем случае это /var/lib/gotosocial/.config/gotosocial/config.yaml

Я его положил в /etc/gotosocial/config.yaml – как системный.

Как узнать домашнюю директорию пользователя gotosocial:

sudo -u gotosocial env | grep HOME
  # HOME=/var/lib/gotosocial
  # SUDO_HOME=/home/mark

Устанавливать sqlite3 не обязательно. GoToSocial использует SQLite library (libsqlite3).

Т.е. Go binary уже включает поддержку SQLite через Go library.

sqlite3 понадобится только если я захожу вручную смотреть базу данных. А так впринципе можно добавлять юзеров и управлять базой данных через CLI GoToSocial.

Первый запуск / проверка (с явным указанием конфига):

sudo -u gotosocial gotosocial \
  --config-path /etc/gotosocial/config.yaml \
  server start

# Если всё нормально, то создастся база данных ...
# ... и не будет ошибок

# Но скорее всего мы получим ошибку из-за того, что
# GoToSocial не может найти шаблоны веб-интерфейса
# Он их ищет в /var/lib/gotosocial/web/template/

# Они лежал в исходниках в папке web/
# Копируем их:
sudo mkdir -p web
sudo cp -r /home/mark/Downloads/gotosocial/web/template ./
sudo chown -R gotosocial:gotosocial ./web

# Теперь можно запускать сервер и должна создастся база данных
# /var/lib/gotosocial/gotosocial.db, кот. указана в /etc/gotosocial/config.yaml

Опять запускаю:


sudo -u gotosocial gotosocial \
  --config-path /etc/gotosocial/config.yaml \
  server start

  # Сервер запустился.
  # И создалась база данных /var/lib/gotosocial/gotosocial.db
  # http:// gts.test / https://gts.test

  # Но на страничке всё отображается бесструктурно.etc
  # GoToSocial ругается на то, что не может прочитать темы из
  # /var/lib/gotosocial/web/assets/themes
  # Давайте эту папку тоже скопируем:

cd /var/lib/gotosocial/web/
sudo cp -r ~/Downloads/gotosocial/web/assets/ ./

  # Теперь запускаем GoToSocial
  # И на этот раз на сайте отображаются картинки
  # Но структуры по-прежнему нет
  # GoToSocial ищет стили в /var/lib/gotosocial/web/assets/dist/base.css
  # но у меня нет папки ./web/dist/ даже в исходниках.
  # Потому что фронтенд не собран.

Фронтенд

В скачанных исходниках в папке web, есть ещё source. Там как раз хранятся исходники, из которых мы должны собрать фронтенд. Если мы их соберём, то создастся нужная нам папка dist.

Готовый фронтенд также можно взять из релиза. Для этого скачиваем релиз со странички: https://codeberg.org/superseriousbusiness/gotosocial/releases

Или через терминал:

# Скачиваем релиз в архиве:
wget https://codeberg.org/superseriousbusiness/gotosocial/releases/download/v0.21.0/gotosocial_0.21.0_linux_amd64.tar.gz

# Или можно скачать через браузер на страничке:
https://codeberg.org/superseriousbusiness/gotosocial/releases

# Распаковывыаем в папку "gts":
mkdir -p gts/
tar -xzf gotosocial_0.21.0_linux_amd64.tar.gz -C gts/

# Заходим в gts/web/assets/ и видим там папку dist
# А также два файлика: all_licenses.txt (список лицензий всех зависимостей) и swagger.yaml (описание API (OpenAPI / Swagger))

# Копируем dist:
sudo cp -r dist/ /var/lib/gotosocial/web/assets/
cd /var/lib/gotosocial/web/assets/
sudo chown -R gotosocial:gotosocial dist/

# И снова запускаем gotosocial:
sudo -u gotosocial gotosocial --config-path /etc/gotosocial/config.yaml server start
  # Теперь всё работает нормально

Как собрать фронтенд

Переходим в папку со скачанными исходниками и находим там web/sources/:

cd ~/Downloads/gotosocial/web/sources/

# Устанавливаем зависимости.
# в папке есть yarn.lock,
# значит лучше использовать yarn:
yarn install

# Сборка:
yarn build

Если всё прошло успешно, то должна появиться папка ../assets/dist. Это и есть наш готовый фронтенд. И его нужно скопировать в /var/lib/gotosocial/web/assets/.

Размер загружаемых файлов

В конфиге за это отвечает настройка media-local-max-size:

# Default: 40MiB (41943040 bytes)
media-local-max-size: 40MiB

Если она не указана, то по умолчанию ограничение: 40MiB

Но так как у нас сервер находится за NGINX'ом, то nginx тоже фильтрует файлы по размеру. В конфиге nginx'а есть настройка client_max_body_size, которая по умолчанию равна 1M. Так что если мы её не указывали, то GoToSocial не сможет загрузить файлы больше 1M.

Поэтому добавим в /etc/nginx/nginx.conf:

# Ставим чуть больше, чем в GoTosocial, ...
# ... чтобы nginx никогда не был узким местом
client_max_body_size 42M;
