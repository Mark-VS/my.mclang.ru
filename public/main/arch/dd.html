dd

Я взял флэшку, на которую хочу записать образ. Для начала неплохо бы её очистить.

Я её вставил. И она у меня автоматически примонтировалась по адресу /run/media/mark/ARCH_202507. Перед тем, как её очищать, её нужно отмонтировать.

sudo umount /dev/sdc1

Теперь сотрём с неё всё.

sudo dd if=/dev/zero of=/dev/sdc bs=1M status=progress
sync
# если нужно извлечь флэшку, то:
sudo eject /dev/sdc

Далее скачиваем образ. Я скачал его в /tmp/archlinux-2025.10.01-x86_64.iso.

Затем запишем образ:

sudo dd if=/tmp/archlinux-2025.10.01-x86_64.iso of=/dev/sdc bs=4M status=progress oflag=sync
sync
sudo eject /dev/sdc

DD (Data Duplicator) - утилита "копирует бит-в-бит".

  • if – input file. Это может быть файл (ISO) или устройство (например, /dev/loop0)
  • of – output file. Диск целиком, а не раздел.
  • bs – block size. Размер блока для чтения/записи.
  • status=progress – Показывает прогресс во время выполнения. Удобно, чтобы видеть, что dd не завис.
  • oflag=sync – флаг, применяемый к выходу. Заставляет dd применять синхронную запись (каждый блок записывается с синхронизацией). Замедляет процесс записи, но повышает надёжность.

Вместо oflag=sync можно использовать conv=fsync.

  • oflag=sync – сбрасывает КЭШ постоянно;
  • conv=fsync – сбрасывает КЭШ один раз в конце записи.
