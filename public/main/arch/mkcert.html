Вернуться назад

mkcert

Для начала установим mkcert:

sudo pacman -S mkcert

Далее:

# создадим локальный CA:
mkcert -install

# создадим сертификат для локального домена gts.test:
mkcert gts.test

В текущей папке появились файлы gts.test.pem и gts.test-key.pem.

Узнать, где CAROOT

Узнать, где находится корневой сертификат:

mkcert -CAROOT
  # /home/mark/.local/share/mkcert

Информация о сертификате:

Посмотреть всю подробную информацию о сертификате можно с помощью openssh:

openssl x509 -in certs/gts.test.pem -text -noout

Хранилища сертификатов

При установке локального сертификата можно встретится с такой проблемой: Firefox и другие firefox-based браузеры как будто его не видят и всё равно воспринимают сайт как небезопасный.

В чём проблема?

Chromium-браузеры (Chrome, Brave и т.д.) используют системное хранилище сертификатов. mkcert автоматически добавляет туда свой root CA. Поэтому всё сразу работает.

Firefox-браузеры (Firefox, LibreWolf, Zen Browser и т.д.) используют собственное хранилище (NSS). Они не доверяют системным хранилищам по умолчанию. Поэтому наш mkcert CA они не видят.

Что с этим можно сделать:

  • Можно добавить свой root CA, чтобы Firefox начал доверять моему локальному сертификату. Для этого нужно зайти сюда:

    Settings → Privacy & Security → Certificates → Manage Certificates → Authorities

    Нажимаем кнопку Import и добавляем наш root CA. В окне с галочками отметим: Trust this CA to identify websites.

    ВАЖНО! Чтобы в окне импорта отображались скрытые файлы нужно нажать Ctrl+H. У меня rootCA.pem находится в ~/.local/share/mkcert/rootCA.pem (то есть, внутри скрытой папки .local). И если я не нажму Ctrl+H то в окне выбора файлов не будут отображаться файлы внутри скрытой папки, потому что всё то, что находится внутри скрытой папки по умолчанию счатается тоже скрытым. Окно выбора просто не увидит файл rootCA.pem.

    Теперь во вкладке Authorities появился мой центр сертификации, который называется mkcert development CA.

  • Можно включить использование системного хранилища. Для этого открываем:

    about:config

    Находим параметр security.enterprise_roots.enabled и выставляем его значение в true
